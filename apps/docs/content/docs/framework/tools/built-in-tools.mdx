ADK-TS provides a collection of pre-built tools that handle common agent tasks. These tools are tested, optimized, and ready to use without custom development. Simply import the tool you need and add it to your agent's tools list.

## Quick Start

Using built-in tools takes just three steps:

```typescript
import { LlmAgent, GoogleSearch, HttpRequestTool } from "@iqai/adk";

const agent = new LlmAgent({
  name: "researchAgent",
  instruction: "Help users research topics using web search and HTTP requests",
  tools: [new GoogleSearch(), new HttpRequestTool()],
});
```

That's it! The agent now has search and HTTP capabilities.

## Available Tools

ADK-TS includes these built-in tools:

| Tool                    | Purpose                         | Requires Config |
| ----------------------- | ------------------------------- | --------------- |
| **GoogleSearch**        | Search the web for information  | No              |
| **HttpRequestTool**     | Make HTTP requests to APIs      | No              |
| **FileOperationsTool**  | Read/write files on disk        | Optional        |
| **UserInteractionTool** | Prompt users for input          | No              |
| **LoadMemoryTool**      | Search conversation history     | No              |
| **LoadArtifactsTool**   | Access uploaded files           | No              |
| **GetUserChoiceTool**   | Present multiple choice options | No              |
| **TransferToAgentTool** | Hand off to another agent       | No              |
| **ExitLoopTool**        | Exit workflow loops             | No              |

## Google Search Tool

Search the web for real-time information.

**Import:**

```typescript
import { GoogleSearch } from "@iqai/adk";
```

**Setup:**

```typescript
const agent = new LlmAgent({
  name: "researchAgent",
  instruction: "Help users research topics by searching the web",
  tools: [new GoogleSearch()],
});
```

**Tool Parameters:**

| Parameter     | Type   | Required | Description                                       |
| ------------- | ------ | -------- | ------------------------------------------------- |
| `query`       | string | Yes      | The search query to execute                       |
| `num_results` | number | No       | Number of results to return (default: 5, max: 10) |

**How It's Triggered:**

The agent automatically uses this tool when the user asks about current information:

```typescript
// User: "What are the latest updates about AI in 2024?"
// Agent internally decides it needs current information, so it calls:
// google_search({ query: "latest AI updates 2024", num_results: 5 })
// Agent receives search results and summarizes them for the user
```

**Capabilities:**

- Real-time web search results
- Configurable number of results
- LLM-friendly response formatting

<Callout type="info" title="Current Status">
  The current GoogleSearch implementation returns mock results. Production use
  requires integration with actual Google Search API and proper credentials.
</Callout>

## HTTP Request Tool

Make HTTP requests to APIs and web services.

**Import:**

```typescript
import { HttpRequestTool } from "@iqai/adk";
```

**Setup:**

```typescript
const agent = new LlmAgent({
  name: "apiAgent",
  instruction: "Fetch data from APIs and integrate external services",
  tools: [new HttpRequestTool()],
});
```

**Tool Parameters:**

| Parameter | Type   | Required | Description                                                              |
| --------- | ------ | -------- | ------------------------------------------------------------------------ |
| `url`     | string | Yes      | The URL to request                                                       |
| `method`  | string | No       | HTTP method (GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS). Default: GET |
| `headers` | object | No       | Custom headers (e.g., `{ "Authorization": "Bearer token" }`)             |
| `body`    | string | No       | Request body (for POST, PUT, PATCH)                                      |

**How It's Triggered:**

The agent uses this tool when it needs to call an external API:

```typescript
// User: "Get the weather from the API"
// Agent calls:
// http_request({
//   url: "https://api.weather.com/forecast?city=london",
//   method: "GET",
//   headers: { "Authorization": "Bearer YOUR_API_KEY" }
// })
// Agent receives the response and processes it
```

**Real-World Example:**

```typescript
const agent = new LlmAgent({
  name: "weatherBot",
  instruction: `Help users get weather information. You can fetch weather data 
               from the weather API using the http_request tool.`,
  tools: [new HttpRequestTool()],
});

// User: "What's the weather in Paris?"
// Agent fetches from API and tells user: "It's 15Â°C and cloudy in Paris"
```

**Capabilities:**

- All HTTP methods: GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS
- Custom headers for authentication
- JSON and form data support
- Response parsing

**Common Use Cases:**

- REST API integration
- Third-party service calls
- Data fetching and submission
- Webhook communication

## File Operations Tool

Read, write, and manage files on disk.

**Import:**

```typescript
import { FileOperationsTool } from "@iqai/adk";
```

**Setup:**

```typescript
const agent = new LlmAgent({
  name: "fileAgent",
  instruction:
    "Help users read and write files. You can read any file and write results.",
  tools: [
    new FileOperationsTool({
      basePath: "/safe/directory", // Optional, defaults to cwd
    }),
  ],
});
```

**Tool Parameters:**

| Parameter   | Type   | Required | Description                                                                          |
| ----------- | ------ | -------- | ------------------------------------------------------------------------------------ |
| `operation` | string | Yes      | Operation to perform: `read`, `write`, `append`, `delete`, `exists`, `list`, `mkdir` |
| `path`      | string | Yes      | File or directory path (relative to `basePath`)                                      |
| `content`   | string | No       | Content to write (required for `write` and `append`)                                 |

**How It's Triggered:**

```typescript
// User: "Save this data to a file called results.txt"
// Agent calls:
// file_operations({
//   operation: "write",
//   path: "results.txt",
//   content: "The data to save"
// })
// Agent: "Saved data to results.txt"

// User: "Read the contents of config.json"
// Agent calls:
// file_operations({
//   operation: "read",
//   path: "config.json"
// })
// Agent reads and displays the content
```

**Supported Operations:**

- `read` - Read file contents
- `write` - Write or overwrite files
- `append` - Add content to existing files
- `delete` - Remove files
- `exists` - Check if file exists
- `list` - List directory contents
- `mkdir` - Create directories

**Security:**

- Limited to specified `basePath` directory
- Prevents path traversal attacks
- Validates all file operations

## User Interaction Tool

Prompt users for input during agent execution.

**Import:**

```typescript
import { UserInteractionTool } from "@iqai/adk";
```

**Setup:**

```typescript
const agent = new LlmAgent({
  name: "approvalAgent",
  instruction: `You help users with approvals. When you need user input or confirmation, 
               use the user_interaction tool with a clear prompt. You can present multiple 
               choice options if applicable.`,
  tools: [new UserInteractionTool()],
});
```

**Tool Parameters:**

The LLM calls this tool with these parameters:

| Parameter      | Type             | Required | Description                                    |
| -------------- | ---------------- | -------- | ---------------------------------------------- |
| `prompt`       | string           | Yes      | The question or message to display to the user |
| `options`      | array of strings | No       | Multiple choice options to present to the user |
| `defaultValue` | string           | No       | Default value if user provides no input        |

**How It's Triggered:**

The agent triggers this tool when it needs user input. For example:

```typescript
// User: "I want to delete my account"
// Agent internally decides it needs confirmation, so it calls user_interaction:

// The agent LLM generates a call like:
// user_interaction({
//   prompt: "Are you absolutely sure you want to delete your account? This cannot be undone.",
//   options: ["Yes, delete it", "No, keep it"]
// })

// User sees: "Are you absolutely sure you want to delete your account? This cannot be undone?"
//            1. Yes, delete it
//            2. No, keep it
// User chooses: 1
// Agent receives: "Yes, delete it" and proceeds with deletion
```

**Real-World Examples:**

```typescript
// Example 1: Simple confirmation
const agent = new LlmAgent({
  name: "paymentAgent",
  instruction: `Help users process payments. Always confirm the payment amount with 
               the user before processing. Use user_interaction to get confirmation.`,
  tools: [new UserInteractionTool()],
});

// User: "Process payment of $100"
// Agent: "Please confirm: Do you want to process a $100 payment?"
// User responds and agent proceeds

// Example 2: Multiple choice selection
const agent = new LlmAgent({
  name: "subscriptionAgent",
  instruction: `Help users choose subscription plans. When they want to subscribe, 
               present the available options using user_interaction and ask them to choose.`,
  tools: [new UserInteractionTool()],
});

// User: "I want to upgrade my subscription"
// Agent: "Which plan would you like?"
//        1. Basic - $9/month
//        2. Pro - $29/month
//        3. Enterprise - $99/month
// User selects option 2
// Agent: "Great! You've selected the Pro plan"

// Example 3: With default value
const agent = new LlmAgent({
  name: "settingsAgent",
  instruction: `Help users configure their preferences. Use user_interaction to get 
               their preferences with sensible defaults.`,
  tools: [new UserInteractionTool()],
});

// User: "Change my notification frequency"
// Agent: "How often should we send notifications? (default: daily)"
// User just presses enter
// Agent uses default value: "daily"
```

**Common Use Cases:**

- Get approval for critical actions
- Present multiple options and let user choose
- Confirm important operations before execution
- Collect user preferences
- Multi-step workflows requiring decisions

<Callout type="info" title="Long-Running by Nature">
  The UserInteractionTool is marked as `isLongRunning: true` because it waits
  for user input. The agent will pause execution until the user responds.
</Callout>

## Load Memory Tool

Search conversation history and past interactions.

**Import:**

```typescript
import { LoadMemoryTool } from "@iqai/adk";
```

**Setup:**

```typescript
const agent = new LlmAgent({
  name: "contextAgent",
  instruction: "Remember past interactions and reference them when relevant",
  tools: [new LoadMemoryTool()],
});
```

**Tool Parameters:**

| Parameter | Type   | Required | Description                                               |
| --------- | ------ | -------- | --------------------------------------------------------- |
| `query`   | string | Yes      | What to search for in memory (e.g., "previous purchases") |

**How It's Triggered:**

The agent uses this to recall past information:

```typescript
// User: "What did I ask you about last time?"
// Agent calls:
// load_memory({ query: "previous conversation topics" })
// Agent retrieves relevant memories and responds with the information

// User: "I have that preference set, right?"
// Agent calls:
// load_memory({ query: "user preferences" })
// Agent confirms based on retrieved memory
```

**Capabilities:**

- Search past conversations
- Retrieve user preferences
- Reference previous decisions
- Maintain context across sessions

Access and process uploaded files.

```typescript
import { LoadArtifactsTool } from "@iqai/adk";
```

**Setup:**

```typescript
const agent = new LlmAgent({
  name: "documentAnalyzer",
  instruction: "Analyze uploaded documents and extract information from them",
```

**Tool Parameters:**

| Parameter | Type   | Required | Description                                                      |
| --------- | ------ | -------- | ---------------------------------------------------------------- |

```typescript
// User: "Analyze my resume"
// Agent calls:
// load_artifacts({ query: "resume" })
// Agent retrieves the uploaded resume and analyzes it

// load_artifacts({ query: "contract PDF" })

```

**Capabilities:**

- List uploaded files
- Access file contents
- Automatic format detection
- Support for various file types (PDF, images, text, etc.)

## Specialized Tools

These tools handle specific workflow scenarios.

### Get User Choice Tool

Present users with multiple options and collect their selection.

```typescript
import { GetUserChoiceTool } from "@iqai/adk";

const agent = new LlmAgent({
  tools: [new GetUserChoiceTool()],
});

// Agent can now present options like "Choose: A, B, or C?"
```

### Transfer to Agent Tool

Hand off work to another specialized agent.

```typescript
import { TransferToAgentTool } from "@iqai/adk";

const agent = new LlmAgent({
  tools: [new TransferToAgentTool()],
});

// Agent can transfer to a billing_agent, support_agent, etc.
```

### Exit Loop Tool

Exit from workflow loops when done.

```typescript
import { ExitLoopTool } from "@iqai/adk";

const agent = new LlmAgent({
  tools: [new ExitLoopTool()],
});

// Agent can break out of loops when completion criteria are met
```

## Common Patterns

### Information Gathering Agent

Combine search and memory tools for research:

```typescript
const researchAgent = new LlmAgent({
  name: "researcher",
  instruction:
    "Research topics by searching the web and recalling relevant information",
  tools: [new GoogleSearch(), new LoadMemoryTool()],
});
```

### Document Processing Agent

Work with uploaded files:

```typescript
const documentAgent = new LlmAgent({
  name: "documentProcessor",
  instruction: "Analyze uploaded documents and save results",
  tools: [
    new LoadArtifactsTool(),
    new FileOperationsTool({ basePath: "/output" }),
  ],
});
```

### API Integration Agent

Call external APIs with HTTP tool:

```typescript
const apiAgent = new LlmAgent({
  name: "apiIntegration",
  instruction: "Fetch data from APIs and process it",
  tools: [new HttpRequestTool(), new FileOperationsTool()],
});
```

### Interactive Agent

Get user input during execution:

```typescript
const interactiveAgent = new LlmAgent({
  name: "assistant",
  instruction: "Ask for user confirmation before making changes",
  tools: [new UserInteractionTool(), new FileOperationsTool()],
});
```

## Best Practices

### Tool Selection

Choose tools that match your agent's purpose. Start simple and add tools as needed.

**Good:**

```typescript
// Search-focused agent
const searchAgent = new LlmAgent({
  tools: [new GoogleSearch()],
});
```

**Avoid:**

```typescript
// Don't add tools the agent won't need
const searchAgent = new LlmAgent({
  tools: [
    new GoogleSearch(),
    new FileOperationsTool(),
    new UserInteractionTool(),
    new HttpRequestTool(),
    // Too many tools confuse the LLM
  ],
});
```

### Error Handling

Built-in tools handle common errors gracefully:

- **Network failures**: HTTP and Search tools retry automatically
- **Invalid files**: FileOperationsTool returns clear error messages
- **User cancellation**: UserInteractionTool handles cancellations
- **API errors**: All tools provide meaningful error responses

### Security Considerations

- **File Operations**: Always set a safe `basePath` to prevent unauthorized access
- **HTTP Requests**: Be cautious with authentication headers in logs
- **User Interaction**: Don't expose sensitive data in prompts
- **External APIs**: Use proper credentials and authentication tokens

<Callout type="warn" title="External Service Dependencies">
  Tools that call external services (GoogleSearch, HttpRequestTool) depend on
  network availability and external service reliability. Design your agents to
  handle failures gracefully.
</Callout>
