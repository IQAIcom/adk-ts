The Langfuse Plugin integrates `@iqai/adk` agents with [Langfuse](https://langfuse.com/docs) for robust tracing, monitoring, and observability of LLM and tool executions. It automatically records user messages, agent actions, model generations, and tool calls, including errors and token usage.

This plugin is especially useful for:

- **Observability** of agent workflows
- **Debugging** tool or model errors
- **Tracking** model usage and token consumption
- **Correlating** events across complex agent hierarchies

## Key Features

- **Automatic tracing**: Tracks every invocation, agent, tool, and model generation
- **Error monitoring**: Captures LLM or tool errors with full context
- **Token and model usage tracking**: Aggregates input/output tokens and models used per invocation
- **Hierarchical spans**: Supports nested agents and tool calls with parent-child relationships
- **Flexible serialization**: Converts complex `Content` and `Event` objects into structured logs
- **Flush and shutdown control**: Manually flush or shutdown Langfuse client if needed

## Installation

```bash
pnpm add @iqai/adk langfuse
```

## Quick Start

```typescript
import { AgentBuilder, LangfusePlugin } from "@iqai/adk";

const { runner } = await AgentBuilder.withModel("gpt-4o")
  .withPlugins(
    new LangfusePlugin({
      publicKey: process.env.LANGFUSE_PUBLIC_KEY!,
      secretKey: process.env.LANGFUSE_SECRET_KEY!,
    }),
  )
  .build();
```

## Configuration

| Option          | Type   | Default                           | Description                     |
| --------------- | ------ | --------------------------------- | ------------------------------- |
| `name`          | string | `"langfuse_plugin"`               | Plugin name for identification  |
| `publicKey`     | string | -                                 | Langfuse public API key         |
| `secretKey`     | string | -                                 | Langfuse secret API key         |
| `baseUrl`       | string | `"https://us.cloud.langfuse.com"` | Langfuse API endpoint           |
| `release`       | string | undefined                         | Optional release tag for traces |
| `flushAt`       | number | `1`                               | Flush after this many events    |
| `flushInterval` | number | `1000`                            | Flush interval in milliseconds  |

### Configuration Details

- **`publicKey` / `secretKey`** _(required)_
  - Langfuse API credentials
  - Always store in environment variables, never hardcode

- **`baseUrl`** _(default: `"https://us.cloud.langfuse.com"`)_
  - Set to `"https://eu.cloud.langfuse.com"` for EU region
  - Or use a self-hosted Langfuse instance URL

- **`release`** _(optional)_
  - Tags all traces with a release version (e.g., `"v1.2.3"`)
  - Useful for tracking performance across deployments

- **`flushAt`** _(default: 1)_
  - Number of events to buffer before flushing to Langfuse
  - Higher values reduce API calls but increase delivery delay
  - Use `1` for development (immediate), `10+` for production (batched)

- **`flushInterval`** _(default: 1000)_
  - Maximum time in ms between flushes
  - Works alongside `flushAt` — whichever triggers first

## Callback Hooks

The plugin intercepts key agent events to send structured logs to Langfuse:

### User messages

- **`onUserMessageCallback`**: Records user input and triggers a `user_message` event

### Agent execution

- **`beforeAgentCallback` / `afterAgentCallback`**: Starts and ends agent spans. Tracks input, output, models used, and sub-agent hierarchy

### Model calls

- **`beforeModelCallback` / `afterModelCallback`**: Captures model request and response, including token usage and generation metadata
- **`onModelErrorCallback`**: Logs errors for LLM calls with full context

### Tool calls

- **`beforeToolCallback` / `afterToolCallback`**: Captures tool invocation and results
- **`onToolErrorCallback`**: Records tool failures and metadata

### Run lifecycle

- **`beforeRunCallback` / `afterRunCallback`**: Tracks overall invocation start and completion
- **`onEventCallback`**: Captures all events in the invocation, including function calls and final outputs

## Advanced Usage

### Customizing Serialization

`LangfusePlugin` converts complex `Content` or `Event` objects to structured logs. You can extend:

```typescript
class CustomLangfusePlugin extends LangfusePlugin {
  toPlainText(data: any): string {
    // Custom logic to extract plain text
    return super.toPlainText(data);
  }
}
```

### Manual Flush and Shutdown

By default, the Langfuse client batches events and flushes automatically based on `flushAt` and `flushInterval`. For explicit control:

#### `flush()`

```typescript
await langfusePlugin.flush();
```

Forces all buffered data to be sent immediately. Use in short-lived environments (serverless, scripts, cron jobs). Does **not** shut down the client — the plugin can continue recording events.

#### `close()`

```typescript
await langfusePlugin.close();
```

Gracefully shuts down the Langfuse client after flushing remaining data. Use when your application is terminating. Once called, the plugin should be considered inactive.

#### Recommended Pattern

In short-lived processes:

```typescript
await langfusePlugin.flush();
await langfusePlugin.close();
```

In long-running applications (servers), you typically don't need to call either method manually.

### Environment-specific Configuration

```typescript
import { LangfusePlugin } from "@iqai/adk";

const getLangfuseConfig = () => {
  if (process.env.NODE_ENV === "production") {
    return {
      publicKey: process.env.LANGFUSE_PUBLIC_KEY!,
      secretKey: process.env.LANGFUSE_SECRET_KEY!,
      baseUrl: process.env.LANGFUSE_BASE_URL || "https://us.cloud.langfuse.com",
      flushAt: 10,
      flushInterval: 5000,
    };
  }
  return {
    publicKey: process.env.LANGFUSE_PUBLIC_KEY!,
    secretKey: process.env.LANGFUSE_SECRET_KEY!,
    flushAt: 1,
    flushInterval: 1000,
  };
};

const langfusePlugin = new LangfusePlugin(getLangfuseConfig());
```

### With Multiple Plugins

```typescript
import { InMemoryRunner, LLMRegistry } from "@iqai/adk";
import {
  LangfusePlugin,
  ToolOutputFilterPlugin,
  ReflectAndRetryToolPlugin,
} from "@iqai/adk";

const runner = new InMemoryRunner(agent, {
  appName: "production-app",
  plugins: [
    new LangfusePlugin({
      publicKey: process.env.LANGFUSE_PUBLIC_KEY!,
      secretKey: process.env.LANGFUSE_SECRET_KEY!,
    }),
    new ToolOutputFilterPlugin({
      filterModel: LLMRegistry.newLLM("gemini-1.5-flash"),
    }),
    new ReflectAndRetryToolPlugin({ maxRetries: 3 }),
  ],
});
```

## Best Practices

### Flush Settings

- Use `flushAt: 1` and `flushInterval: 1000` for development (immediate visibility)
- Use `flushAt: 10` and `flushInterval: 5000` for production (reduced API calls)

### Release Tags

- Set meaningful `release` tags (e.g., git SHA or semantic version)
- Helps track performance regressions across deployments

### Region Configuration

- Use `baseUrl: "https://eu.cloud.langfuse.com"` for EU data residency requirements
- Or point to a self-hosted instance

## Troubleshooting

**No data appearing in Langfuse:**

- Verify API keys are correct and for the right project
- Check network connectivity to Langfuse
- Ensure plugin is registered on the Runner/AgentBuilder/LlmAgent
- Check `flushAt` isn't too high (data may be buffered)

**Missing traces or spans:**

- Ensure proper shutdown: call `flush()` then `close()` in short-lived processes
- Check for errors in plugin execution (wrapped errors surface in agent output)

**Performance issues:**

- Increase `flushInterval` for better batching
- Increase `flushAt` to reduce API call frequency
- Check network latency to Langfuse API

**Authentication errors:**

- Verify `publicKey` and `secretKey` are correct
- Check API key permissions in Langfuse dashboard
- Ensure keys are for the correct project and environment
