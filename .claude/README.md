# ADK-TS Claude Code Configuration

This directory contains project-scoped skills, agents, and commands for Claude Code. Everything here is committed to the repo so all team members get them automatically.

## Skills (Auto-Triggered)

Skills activate automatically when Claude detects matching context — no manual invocation needed.

| Skill                 | Triggers When                                               | What It Does                                                                                |
| --------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **adk-style-guide**   | Writing any docs, READMEs, changelogs, or technical content | Enforces ADK-TS brand voice, terminology, canonical URLs, and SEO keywords                  |
| **adk-docs-writer**   | Creating/editing MDX pages in `apps/docs/content/docs/`     | Knows Fumadocs components, page archetypes, navigation, SEO, and content placement rules    |
| **adk-readme-writer** | Creating/editing any `README.md` in the monorepo            | Knows the four branded README archetypes (root, package, app/contributor, starter template) |

## Agents (Autonomous Workers)

Agents run in isolated context and can be invoked by Claude when matching tasks arise, or spawned manually.

| Agent                      | Model   | What It Does                                                                                            |
| -------------------------- | ------- | ------------------------------------------------------------------------------------------------------- |
| **docs-writer**            | inherit | Writes and updates documentation pages — reads source code, follows archetypes, registers in navigation |
| **docs-reviewer**          | sonnet  | Reviews docs for quality, accuracy, brand consistency, and style guide compliance (read-only)           |
| **mcp-docs-generator**     | inherit | Generates complete MCP server doc pages from package source code                                        |
| **docs-freshness-checker** | sonnet  | Background agent that compares docs against source code to find stale content (read-only)               |

## Commands (Manual Invocation)

Invoke these with `/command-name` in Claude Code.

| Command             | What It Does                                                                                                                                                      |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/adk-docs-audit`   | Scans the entire monorepo for documentation inconsistencies (brand, URLs, navigation, frontmatter, READMEs). Reports only — no changes.                           |
| `/adk-starter-sync` | Syncs shared boilerplate content across all starter template READMEs in `apps/starter-templates/*/README.md`. Detects drift and propagates updates.               |
| `/adk-changelog`    | Generates `changelog-entry.md` and `release-notes.md` for polished changelog entries and community announcements. `CHANGELOG.md` is auto-generated by changesets. |

## Quick Reference

- **Brand rule:** Always "ADK-TS" in prose, never bare "ADK", never expand to "Agent Development Kit"
- **Default model:** `gemini-2.5-flash` in code examples
- **Docs site:** `apps/docs/content/docs/` (Fumadocs MDX)
- **MCP servers:** `mcp-servers/iq-ai-servers/` (built-in) and `mcp-servers/third-party-wrappers/` (wrapped)
- **Guides:** `framework/guides/` for cross-cutting content (best practices, troubleshooting, deployment, integrations)
- **Node.js:** `>=22.0`
- **GitHub org casing:** `IQAIcom` (not `IQAICOM`)
